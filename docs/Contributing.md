# Contributing

## Ground Rules

- Follow `docs/00_SOURCE_OF_TRUTH.md`.
- Keep changes scoped to the requested phase or issue.
- Do not add gameplay, APIs, or AI systems outside explicit scope.
- Add tests for implemented behavior.
- Run relevant validation before opening a pull request.

## Pull Request Expectations

- Explain what changed.
- Link related issue or phase.
- Include test commands and results.
- Include screenshots for UI-visible changes when possible.
- Avoid unrelated formatting churn.

## Branch Naming

Use clear branch names:

- `phase-20-portfolio-showcase-release`
- `fix-auth-refresh`
- `docs-deployment-notes`
